# 物品保质期管理系统 - 微信小程序前端 (Expiration Management System - Mini Program)

本项目为物品保质期管理系统的微信小程序客户端。系统采用微信原生框架进行开发，为用户提供直观、便捷的卡片式物品保质期管理工作台。通过与后端 RESTful API 进行数据交互，实现物品全生命周期的时效性监控，帮助用户及时处理临期与过期物品。

### 一、 功能特性

1. 账户认证体系

* 登录状态鉴权：支持用户通过账号密码安全登录，登录成功后获取唯一用户 ID 并进行跨页面数据隔离与会话保持。
* 弹窗注册：采用非跳转的自定义弹窗遮罩（Dialog Overlay）处理新用户注册，优化交互链路。

2. 物品保质期管理

* 动态卡片列表：以平铺卡片（Product Card）形式直观展示物品名称、分类标签及过期具体时间。
* 可视化录入：内置表单弹窗，支持物品名称输入、分类选择（选择器组件）以及过期日期选择，全量对接后端插入接口。
* 异步双重确认删除：删除物品时触发二次确认弹窗，防止用户高危误触，数据同步刷新。

3. 分类快速筛选

* 快捷导航：主面板顶部集成分类过滤器（Filter Buttons）。
* 精准过滤：支持按食物（food）、药品（medicine）、日常用品（daily_necessity）进行条件切换，即时重载列表数据。

### 二、 技术栈与工具

1. 核心框架：微信小程序原生框架 (WXML / WXSS / JavaScript)
2. 网络通信：底层基于原生 wx.request 接口，支持异步通信与前后端数据交互。
3. 布局与样式：全页采用 Flexbox 响应式布局，配合 rpx 相对单位实现多端屏幕分辨率自适应。

### 三、 项目工程结构

```text
pages/
    login/                         # 登录与注册模块
        login.js                   # 登录/注册表单提交、异步请求封装逻辑
        login.wxml                 # 视图层：欢迎面板、输入框、产品说明、注册弹框
        login.wxss                 # 样式层：毛玻璃阴影、绿色主调按钮、遮罩层动画
    products/                      # 物品管理主工作台
        products.js                # 列表加载、类别筛选、增删接口异步回调逻辑
        products.wxml              # 视图层：用户信息、分类按钮组、卡片列表、确认弹窗
        products.wxss              # 样式层：卡片悬浮悬停动画、删除警告色、页面响应式
app.js                             # 小程序全局逻辑入口与生命周期管理
app.json                           # 全局页面路由注册及全局导航栏样式配置
project.config.json                # 微信开发者工具项目基础配置文件

```

### 四、 本地部署与联调指引

1. 导入项目

* 打开微信开发者工具，点击导入项目。
* 选择当前前端项目根目录（确保包含 app.json 文件的层级）。
* AppID 可填入个人开发者 AppID 或直接选择使用测试号。

2. 开启本地域名校验放行 (关键)
由于本地联调调用的是本地后端接口，小程序默认的安全策略会拦截非合法域名的请求，请务必开启以下配置：

* 在微信开发者工具右上角，点击详情按钮。
* 切换到本地设置标签页。
* 找到并勾选：不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书。

3. 后端接口适配

* 确保本地 Spring Boot 后端项目已在 8080 端口启动。
* 小程序内部默认请求基地址已指向：http://localhost:8080/
* 配置完成后，在模拟器中点击重新编译即可查看完整的登录表单与物品列表。
